## 1. Что нужно было сделать

Задача состояла в том, чтобы реализовать набор функций для работы с данными, используя библиотеку `NumPy`. Нужно было уйти от обычных циклов Python к векторизированным операциям, которые работают намного быстрее.

Основные этапы:
1.  Создать и преобразовать массивы (векторы и матрицы).
2.  Выполнить базовую линейную алгебру (умножение, определители, решение систем уравнений).
3.  Загрузить реальные данные (оценки студентов) из CSV-файла.
4.  Посчитать статистику и нормализовать данные.
5.  Визуализировать результаты с помощью `matplotlib` и `seaborn`.

## 2. Как я это реализовал

Весь код разбит на логические блоки. Главное правило, которого я придерживался — **никаких циклов `for`** там, где можно использовать встроенные функции NumPy.

### Работа с массивами
Для создания векторов использовал `np.arange`, а для случайных матриц — `np.random.rand`. 
Один из важных моментов изменение формы массива (`reshape`). Убедился, что общее количество элементов сохраняется, иначе NumPy выдает ошибку. Транспонирование сделал через свойство `.T`, это самый быстрый и читаемый способ.

### Линейная алгебра
Здесь я использовал модуль `numpy.linalg`. 
*   Для решения системы уравнений $Ax = b$ я применил `np.linalg.solve`. Это лучше, чем искать обратную матрицу вручную, потому что так точнее и быстрее.
*   Проверку обратной матрицы делал через умножение исходной матрицы на полученную — результат должен быть единичной матрицей (`np.eye`).

### Анализ данных
Данные загружал через `pandas`, но сразу конвертировал в `numpy` массив методом `.to_numpy()`, чтобы дальше работать только с NumPy.
*   **Статистика:** Считал среднее, медиану и перцентили. Медиана важна, потому что она меньше зависит от выбросов (например, если один студент получил 0, среднее сильно упадет, а медиана останется устойчивой).
*   **Нормализация:** Использовал Min-Max scaling. Данные приводятся к диапазону от 0 до 1. Это нужно, чтобы признаки с разными масштабами (например, рост в см и вес в кг) имели равный вклад при обучении моделей в будущем.

### Графики
Функции визуализации автоматически сохраняют картинки в папку `plots/`. 
*   Гистограмма показывает распределение оценок.
*   Heatmap (тепловая карта) помогает увидеть корреляцию между предметами (например, если хорошо знаешь физику, скорее всего, знаешь и математику).

## 3. Нюансы, с которыми столкнулся

1.  **Размерности (Shape):** Самая частая ошибка — `ValueError: shapes not aligned`. При умножении матриц нужно строго следить, чтобы количество столбцов первой матрицы совпадало с количеством строк второй. Я решал это, проверяя `.shape` перед операциями.
2.  **Плавающая точка:** При сравнении результатов в тестах нельзя использовать обычное `==`. Из-за погрешностей вычислений `0.1 + 0.2` не всегда равно `0.3`. Поэтому в тестах я использовал `np.allclose` или округление.
3.  **Деление на ноль:** В функции нормализации есть риск деления на ноль, если все оценки у студентов одинаковые (max == min). Я добавил простую проверку: если разность равна нулю, возвращаем массив нулей.
4.  **Пути к файлам:** Чтобы код работал и у меня, и на сервере проверки, я использовал относительные пути (`data/students_scores.csv`) и создавал папку `plots` программно через `os.makedirs(..., exist_ok=True)`, чтобы не ловить ошибку `FileNotFoundError`.
